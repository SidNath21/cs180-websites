Part 1.1: Finite Difference Operator

The first part of the project involved leveraging the finite difference operators (Dx = [[-1, 1]] & Dy = [[1], [-1]]) to find the partial derivatives of the cameraman image displayed below. By convolving this image with Dx filter, I can visualize the vertical edges in the image. Similarly, convolving the input image with the Dy operator helps illustrate the horizontal edges of my image. After calculating both partial derivatives, I calculated the Gradient Magnitude Image by taking the square root of the sum of the squares of both partial derivative images. Finally, to turn this result into an edge image, I binarized the gradient magnitude image by selecting an appropriate threshold (50) and assigning pixels to be either 0 or 1 depending on their value.

Part 1.2: Derivative of Gaussian (DoG) Filter

Due to the noise of the originial cameraman.png image, I first applied a Gaussian filter to better surpress the noise and capture the edges of the image more clearly. To do this, I constructed a Gaussian Filter with kernel size = 7 and sigma = 1.167 and applied this filter on the cameraman image before applying the finite difference operators from above. I chose this filter size because it was large enough to display the edges in a visually pleasing manner without making the edges appear too big. I also used a threshold of 60 compared to the threshold of 50 I used in the previous part as this threshold gave better results!

These images appear to have less noise than the images generated without the Gaussian filter. This is because the Gaussian filter acts as a low pass filter that helps eliminate some of the high frequency noise in the original image. Since we applied the Gaussian filter, the edges of our image appear thicker allowing them to stand out in the final binarized image. Overall, applying the Gaussian filter provided better results.

To speed up the process and be more computationally efficient, we can utilize the fact that Convolutions are associative. This means we can first apply the finite difference operators on the Gaussian filter to create the Derivative of Gaussian filters (DoGx & DoGy). This operation is much faster because the size of the filters are much smaller than the dimensions of our input image. Now, we only have to convolve our input image once with the DoGx & DoGy filter to get our results.

Part 2.1: Image "Sharpening"

For this next part of the project, I derived the unsharp masking filter which is used to make images appear sharper. The first step is to take our input image and convolve it with the Gaussian filter to generate a blurred version of the image. Since the Gaussian filter acts as a low pass filter, the resulting blurred image only contains the low frequencies of our original image. Then, we can subtract the blurred version from the original image to get the high frequencies of the image. Since images appear sharper with stronger high frequencies, we can add a disproportionate amount of the high frequences back into our original image to make it appear sharper. For the following examples, I used a Gaussian filter with the kernel size = 9 and sigma = 1.5!

Part 2.2: Hybrid Images

The next part of this project involved taking two separate images and generating hybrid images that takes the high frequencies of one image and applies them to the low frequencies of the other image. The reason this images are called "hybrid" is that high frequencies normally dominate our perception when it is available at a close viewing distance while only the low frequencies of the image can be seen from far away. Therefore, blending the high frequency portion of one image with the low-frequency portion of another generates a hybrid image that leads to different interpretations at different viewing distances.

To get the low frequency signal, we can apply a Gaussian filter to our images. Then, to get the high frequency signal, we can simply subtract the blurred image from the original image. Using this technqiue, I got the following results:

Part 2.3: Gaussian and Laplacian Stacks

For this part of the project, I created a function to generate Gaussian and Laplacian stacks for images as these stacks are needed in the next part of the project for Multi-resolution blending.

To create the Gaussian Stack, I firt took the input image and applied a Gaussian filter on it. To create the successive levels of the Gaussian Stack, I simply applied the Gaussian filter at each level without subsampling. This results in a stack of images with decreasing frequency bands. For the following images, I used a Gaussin filter with kernel size = 21 and sigma = 7

To create the Laplacian Stack, I took each pair of consecutive images from the Gaussian Stack and subtracted the lower frequency image from the high frequency image to get sub-band images of high frequencies. I also appended the Laplacian Stack with the last (most blurred) image from the Gaussian Stack. This gives the Laplacian stack a useful property in which we can now colapse all the images in the Laplacian Stack to get back the original input image.
